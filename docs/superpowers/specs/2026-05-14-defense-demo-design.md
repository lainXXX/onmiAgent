# 答辩功能演示设计方案

## 概述

为 AI Agent 系统设计一个 2 分钟答辩功能演示视频，核心目标是在技术评委面前展示系统的三大核心能力：AI对话、Agent任务执行、RAG问答。

## 演示方案

### 时间分配

| 段落 | 时长 | 内容 |
|------|------|------|
| 开场 | 5s | 直接切入主题 |
| AI对话 | 20s | 工具调用 + 实时执行 |
| Agent任务 | 50s | 子Agent协作完成前端任务（重点） |
| RAG问答 | 25s | 文档检索 + 精准回答 |
| 代码分析 | 20s | 代码理解 + Bash验证 |

**总计：120秒（2分钟）**

### 各段落详细设计

#### 开场（0:00 - 0:05）
- 语气：自信、直接
- 话术："我演示系统三个核心能力——AI对话、Agent任务、RAG问答"

#### 第一段：AI对话（0:05 - 0:25）
**目标**：展示自然语言理解 + 工具调用能力

**场景设计**：
- 用户输入："用 p5.js 画一个旋转彩色三角形"
- 展示模型理解意图 → 调用 BashTool 写代码 → 执行返回可视化结果

**关键画面**：
1. 用户输入框输入问题
2. 模型思考过程（tools being called）
3. Bash执行日志输出
4. 浏览器显示旋转的彩色三角形

#### 第二段：Agent任务（0:25 - 1:15）⭐ 重点
**目标**：展示 Agent 自主规划、协作、交付完整任务的能力

**场景设计**：
- 用户输入："写一个精美的粒子动画首页"
- 完整流程：
  1. PLAN Agent 接收任务，规划步骤（输出任务分解步骤）
  2. SubAgent 协作创建前端项目（HTML/CSS/JS）
  3. CODE_REVIEWER Agent 审查代码
  4. 最终效果展示（精美的粒子动画首页）

**关键画面**：
1. 消息列表展示 PLAN Agent 规划输出
2. 创建的文件列表（index.html, style.css, main.js）
3. CODE_REVIEWER Agent 审查过程
4. 浏览器打开最终效果（粒子动画）

**技术亮点**：
- 子Agent分工协作（PLAN + CODE_REVIEWER）
- 自主文件创建和执行
- 代码质量审查机制

#### 第三段：RAG问答（1:15 - 1:40）
**目标**：展示文档理解和精准检索能力

**场景设计**：
- 上传一份技术文档（Spring AI 官方文档）
- 用户提问："Spring AI 支持哪些 Embedding Model？"
- 展示：文档切片 → 向量检索 → 精准回答

**关键画面**：
1. 文档上传成功
2. 向量检索结果（相似文档片段）
3. 基于检索内容生成的回答

#### 第四段：代码分析（1:40 - 2:00）
**目标**：展示代码理解和执行验证能力

**场景设计**：
- 用户粘贴一段复杂递归代码
- 提问："这段代码的时间复杂度是多少？"
- 展示：代码解析 + 执行结果验证

**关键画面**：
1. 代码粘贴展示
2. AI 分析输出（时间复杂度分析）
3. Bash 执行验证（输入输出结果）

## 视觉设计建议

### 界面布局
- 浏览器窗口占主画面（80%）
- 左侧或底部小窗展示 Terminal/Bash 执行日志
- 聊天界面作为主导航，切换不同功能演示

### 配色方案
- 深色主题背景（专业感）
- 终端使用深蓝/深绿配色
- 粒子动画使用渐变彩色（蓝→紫→粉）

### 录制建议
- 分辨率：1920x1080
- 鼠标操作适当放慢，方便评委看清
- 关键步骤可添加屏幕箭头标注

## 验收标准

1. 2分钟以内完成全部演示
2. 每个功能有明确的开始和结束节点
3. Agent任务段（50s）充分展示子Agent协作亮点
4. 技术评委能清晰理解系统能力边界
